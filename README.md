# Rocky Linux 패킷 제너레이터 설정 가이드 (pktgen.cfg)

이 설명서는 C 기반 고속 패킷 생성기(`pktgen`)의 통합 설정 파일인 `pktgen.cfg` 파일의 구조와 각 매개변수 설정 방법을 설명합니다. 모든 옵션은 명령행 인자 없이 설정 파일 하나로 완전하게 제어됩니다.

---

## 설정 파일 기본 구조

설정 파일은 INI 포맷(`.cfg` 확장자)을 사용하며, **`[system]`**, **`[user]`**, **`[web]`**, **`[was]`**, **`[db]`**, **`[data]`** 총 6개의 섹션으로 구성됩니다.

```ini
[system]
interface = enp0s3
pps = 0
duration = 1

[user]
ip = 10.0.1.1/24
count = 100

[web]
ip = 10.0.2.1/24
count = 5

[was]
ip = 10.0.3.1/24
count = 2

[db]
ip = 10.0.4.1/24
count = 1

[data]
min_kb = 2.0
max_kb = 4.0
use_thread = true
```

---

## 섹션 및 파라미터 상세 안내

### 1. `[system]` 섹션
전체 패킷 주입 시스템의 인프라 설정을 처리합니다.
- **`interface`** (예: `enp0s3`, `eth0`)
  - 패킷을 강제 송출할 리눅스 네트워크 인터페이스 카드(NIC) 이름입니다. (필수 값)
- **`pps`** (예: `1000`, `0`)
  - 초당 송출할 타겟 패킷 수(Packets Per Second)입니다.
  - `0`으로 지정 시 시스템 및 네트워크 카드가 지원하는 **최대 전송률(Maximum Speed)**로 패킷을 주입합니다.
- **`duration`** (예: `5`, `0`)
  - 시뮬레이션을 실행할 시간(분 단위)입니다.
  - `0`으로 지정 시 시간 제한 없이 **무한 실행**하며, 키보드의 `Ctrl+C` 키를 통해 중단할 수 있습니다.

### 2. 시뮬레이션 노드 섹션 (`[user]`, `[web]`, `[was]`, `[db]`)
각 레이어별 가상 노드의 IP 대역과 가상 대수를 결정합니다.
- **`ip`** (예: `10.0.1.1/24`, `192.168.10.1/16`)
  - 가상 노드가 할당받을 시작 IP 주소(Base IP) 및 서브넷 CIDR 표기입니다. 
  - 슬래시(`/`) 뒷부분의 마스크 정보는 설명용이며, 앞부분의 IP 주소를 시작 주소로 하여 설정한 개수(`count`)만큼 IP 주소와 고유 MAC 주소가 1씩 증가하며 자동 매핑됩니다.
- **`count`** (예: `100`, `5`)
  - 시뮬레이션할 해당 레이어의 가상 장비 개수입니다.

### 3. `[data]` 섹션
송출할 패킷 페이로드의 크기 및 고성능화 멀티스레드 옵션을 세팅합니다.
- **`min_kb`** (예: `0.5`, `2.0`)
  - 전송할 트랜잭션당 순수 데이터(Payload)의 최소 크기(KB 단위)입니다. 소수점 입력이 가능합니다.
- **`max_kb`** (예: `1.4`, `4.0`)
  - 전송할 트랜잭션당 순수 데이터(Payload)의 최대 크기(KB 단위)입니다.
  - **참고**: 이더넷 기본 MTU(1500바이트)보다 큰 데이터 크기(예: 2.0KB ~ 4.0KB)가 지정되면 프로그램이 내부적으로 **1,400바이트 단위로 데이터를 쪼개어 연속된 TCP 세그먼트 패킷 흐름으로 분할 송출**합니다.
  - 최대 설정 크기 상한선은 **`10000.0` KB (10MB)** 로 제한되어 있으며, 이를 초과할 경우 10MB로 자동 조정됩니다.
- **`use_thread`** (예: `true`, `false` / `1`, `0`)
  - 병렬 멀티스레드 전송 가동 여부입니다.
  - **`true` (또는 `1`)**: 스레드 모드가 켜집니다. 3개의 전용 스레드(User<->Web, Web<->WAS, WAS<->DB)가 멀티코어 연산을 통해 병렬 주입하여 단일 스레드 대비 **약 2배 이상의 PPS 가속 성능**을 냅니다.
  - **`false` (또는 `0`)**: 스레드 모드가 꺼집니다. 싱글스레드 순차 방식으로 요청-응답 트랜잭션 흐름 순서가 정연하게 맞춰진 상태로 전송됩니다.

## 실행 방법

모든 인자 설정이 `pktgen.cfg` 파일에 들어있으므로, 프로그램 실행 경로에 해당 설정 파일이 존재한다면 추가 옵션 없이 단순 호출만으로 가동됩니다.

```bash
# 1. 빌드 컴파일
make

# 2. 실행 (디렉토리 내 pktgen.cfg 자동 탐색 로드)
sudo ./pktgen

# 3. 임시로 다른 설정 파일을 불러올 경우 (-c 옵션 이용)
sudo ./pktgen -c my_test_env.cfg
```

종료 시점에는 실행된 명령어 옵션과 Flow별 통계 기록이 종료 시간 포맷 폴더(`YYYYMMDD-HHMMSS`)에 `pktgen_summary.log` 및 `ip.log` 파일로 보관됩니다.
